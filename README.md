## 氣泡排序法 (Bubble Sort)

### 簡述原理
重複比較兩兩相鄰的元素，如果順序錯誤就交換，將大的元素慢慢搬到陣列末尾。

### 複雜度分析
- 平均時間複雜度: O(n²)
- 最壞時間複雜度: O(n²)
- 最好時間複雜度: O(n) (當陣列已排序時)
- 空間複雜度: O(1)

### 模擬實作內容
氣泡排序在數據量增大時，執行時間呈現劇烈的二次方增長。

## 選擇排序法 (Selection Sort)

### 簡述原理
在陣列中找到最小的元素，跟第一個元素交換；然後在剩下的元素中再找最小的，跟第二個交換，依此類推。

### 複雜度分析
- 平均時間複雜度: O(n²)
- 最壞時間複雜度: O(n²)
- 最好時間複雜度: O(n²)
- 空間複雜度: O(1)

### 模擬實作內容
選擇排序的特性是無論數據狀態如何，其比較次數固定，因此三種情況耗時接近。

## 插入排序法 (Insertion Sort)

### 簡述原理
像整理撲克牌一樣，將元素一個一個插入到已經排序好的序列中。

### 複雜度分析
- 平均時間複雜度: O(n²)
- 最壞時間複雜度: O(n²)
- 最好時間複雜度: O(n) (當陣列已接近排序時)
- 空間複雜度: O(1)

### 模擬實作內容
插入排序在「已排序」狀態下具有極佳效能，但在隨機與倒序情況下仍屬二次方成長。

## 合併排序法 (Merge Sort)

### 簡述原理
採用分而治之法，先遞迴地將陣列分割成兩半，排序完成後再合併起來。

### 複雜度分析
- 平均時間複雜度: O(n log n)
- 最壞時間複雜度: O(n log n)
- 最好時間複雜度: O(n log n)
- 空間複雜度: O(n)

### 模擬實作內容
合併排序表現非常穩定，數據量增加十倍時，耗時僅增加約十餘倍，而非百倍。

## 快速排序法 (Quick Sort)

### 簡述原理
選擇一個元素作為基準 (pivot)，將陣列分成大於和和小於基準的兩部分，再遞迴地對兩部分進行排序。

### 複雜度分析
- 平均時間複雜度: O(n log n)
- 最壞時間複雜度: O(n²) (當基準選擇極不平衡時)
- 最好時間複雜度: O(n log n)
- 空間複雜度: O(log n) (由於遞迴堆疊)

### 模擬實作內容
快速排序在大多數情況下是實測最快的，但在倒序且選取末端為基準時會明顯變慢。

## 綜合比較
綜合比較來看, 氣泡、選擇和插入排序法在平均和最壞情況下都是 O(n²) 的時間複雜度, 效率較低, 適合小型資料,優點是空間複雜度低, 且氣泡和插入排序法是穩定的。

合併和快速排序法則速度較快, 平均可達 O(n log n), 適合大型資料, 但合併排序法需要額外空間, 快速排序法在最壞情況下會退化, 且通常是不穩定的。
